Why is Programming hard?
1) Computers are dumb

Why is Programming hard?: Teacher Notes
Computers do not have intelligence, they will follow our instructions exactly
When we talk to each other, we have a common context by which we communicate.  Our language is full of ambiguities so we have to make assumptions about what the other speaker means.  Computers do not make assumptions
Human language is imprecise.  Computer language is not
The hard part of programming is not learning the language, it's being able to critically analyze all the contexts that we take for granted when we communicate to each other
One of the most important things to remember is that COMPUTERS CAN ONLY DO ONE THING AT A TIME

Problem solving framework: Teacher Notes
So how do we start to think like a computer?
A lot of the problems we think about doing are actually lots of little problems put together
Because computers can only do one thing at a time, we need to break down our problems as small as possible
Scary word alert: Recursion
Recursion means repeating the same process with smaller piece of the original problem
We stop recursing when the problem is simple enough to solve
Scary word alert: Recursion: Teacher Notes
What I mean by recursion here in our framework is that we will use the same framework on each of the subproblems until they are simple enough that we know how to tell the computer how to do them
Problem solving framework
So our framework really looks like this:
1) Define the problem
2) List the subproblems

- if one subproblem:
  3) solve

- if multiple subproblems
  3.1) recurse on subproblems
  3.2) combine subsolutions

What is abstraction?: Teacher Notes

            

              
One type of abstraction is a name: "Tim"; is a person. We use the name "Tim" as an abstraction of the person Tim. That way, when I want to tell you about Tim, I can just say his name instead of going and getting Tim and pointing at him while I talk
